Image-Guided Needle Insertion Planning

A self-study portfolio project combining medical visualization software with a full systems-engineering description of the same workflow.

New to these concepts? This page is the roadmap/timeline view. For a module-by-module walkthrough that explains each concept before the hands-on steps, see the learning guide.

1. The aim

You're building one coherent project, attacked from two directions:

  • Build it: a small 3D Slicer module that helps plan a percutaneous needle insertion (e.g. a liver biopsy or tumour ablation) — place an entry point and a target on a CT scan, draw the trajectory between them, and check whether that line crosses a structure it shouldn't (a major vessel, a "no-go" zone).
  • Describe and de-risk it: model that exact same clinical workflow using professional systems-engineering methods — an Arcadia/MBSE model in Capella, an IREB-style requirements specification with traceability, and an ISO 14971-style hazard analysis.
Why this combination? A working tool proves you can code in the medical-imaging domain. The SE artifacts prove you think rigorously about why the tool needs to do what it does, and what could go wrong if it doesn't — which is exactly the mindset that matters for interventional/medical research groups.

2. The two halves explained

2.1 The technical half: 3D Slicer + Python (+ VTK/ITK, optionally MONAI)

3D Slicer is a free, open-source desktop application for medical image analysis and visualization. It is built on top of two lower-level C++ libraries:

  • VTK (Visualization Toolkit) — rendering, meshes, volume rendering, 3D scenes.
  • ITK (Insight Toolkit) — image processing and segmentation algorithms (filters, registration, marching cubes, etc.).

Slicer ships with its own bundled Python 3.9 interpreter and exposes the entire VTK/ITK API plus its own data model (vtkMRMLScene, volumes, segmentations, fiducials/markups, models). You write a "scripted module" — a Python file with a small amount of boilerplate — and it appears as a panel in the Slicer UI, with full access to whatever volume/segmentation the user has loaded.

For this project the module will:

  1. Load a CT volume (from a public dataset) and a segmentation of an organ + a vessel.
  2. Let the user place two fiducial points: entry and target.
  3. Draw a line (the planned needle trajectory) between them.
  4. Check whether that line intersects the vessel segmentation.
  5. Show a clear pass/fail result in the UI.

Optional stretch (MONAI/PyTorch): MONAI is a PyTorch-based framework for medical imaging deep learning. The stretch goal is to train a small U-Net on a public segmentation dataset (e.g. liver or hepatic vessels from the Medical Segmentation Decathlon), then load the resulting segmentation into the Slicer module above — bridging "deep learning" and "visualization/planning" in one pipeline.

2.2 The systems-engineering half: Capella/Arcadia, IREB, ISO 14971

MBSE Capella is a free, open-source Model-Based Systems Engineering (MBSE) tool originally developed by Thales, built on Eclipse. It implements the Arcadia method, which structures a system description into four layers:

LayerQuestion it answersFor this project
Operational analysisWhat does the clinician/user need to do, and why? "Interventional radiologist needs to reach a target lesion without damaging major vessels."
System analysisWhat must the system (tool) do to support that? Functions: acquire/load image, let user define entry & target, compute trajectory, verify clearance, display result.
Logical architectureHow is that broken into logical components? Image Loader, Trajectory Planner, Clearance Checker, Viewer.
Physical architectureWhat are the concrete components/interfaces? Slicer scripted module, vtkMRML scene, segmentation node, markups node.

Capella has a built-in "Arcadia method" guide and an in-tool tutorial project, so you learn the notation by following along inside the tool itself.

RE IREB CPRE (International Requirements Engineering Board, Certified Professional for Requirements Engineering) publishes a free Foundation Level syllabus describing how to write good requirements: unambiguous, verifiable, each with a stable ID, classified (functional / non-functional / constraint), and traceable to its source. You'll apply this — not take the exam — by writing ~15–25 requirements for the needle-planning tool and a traceability matrix linking each requirement to the Capella function(s) that realize it and the test/check that verifies it.

Risk ISO 14971 is the medical-device risk management standard. Its core artifact is a hazard analysis table:

HazardCauseEffectSeverity / ProbabilityMitigationLinked requirement
Vessel puncturePlanned trajectory crosses vessel mask Internal bleedingHigh / Medium System flags any trajectory intersecting the vessel segmentation before confirmationREQ-0xx

You'll build a small table like this for the 2-3 main hazards of the planning workflow, each row tracing back to a requirement (and, indirectly, to the Capella function it constrains). This is a simplified version of what a real medical-device risk file looks like.

4. Roadmap

Week 1 — Stand up the tooling

  • Install 3D Slicer, run through the official tutorial path.
  • Download one Medical Segmentation Decathlon task (Liver or Hepatic Vessel) for later use.
  • Install Capella, run the in-tool Arcadia tutorial.
  • Push this repo skeleton to GitHub.

📘 Guide: Version control basics, 3D Slicer basics, MBSE with Capella/Arcadia

Weeks 2–3 — Slicer module

  • Build a Python scripted module: load a volume, place two fiducials (entry & target), draw the trajectory line.
  • Load/segment a vessel structure and check the trajectory against it (intersection test).
  • Display a clear pass/fail result in the module UI.

📘 Guide: Building a Slicer Python module

Weeks 3–4 — Capella model

  • Operational analysis: clinician needs to reach a target safely.
  • System analysis: acquire image, plan path, verify clearance, display result.
  • Logical architecture: components for each function.
  • Physical architecture: map components to the actual Slicer module/Python classes.

📘 Guide: MBSE with Capella/Arcadia

Week 4 — Requirements spec + ISO 14971 hazard analysis

  • Write 15–25 IDed, classified requirements.
  • Build a traceability matrix: requirement → Capella function → verification method.
  • Build a hazard table for the 2–3 main risks, each linked to a requirement.

📘 Guide: Requirements engineering (IREB), Risk management (ISO 14971)

Ongoing — VTK/ITK depth + Preim's book

  • Reproduce one technique from Visual Computing for Medicine (e.g. marching cubes surface extraction, or a custom transfer function for volume rendering) inside or alongside the Slicer module.

📘 Guide: VTK/ITK deep dive

Stretch — ML bridge

  • Train a small MONAI U-Net on a Decathlon task.
