## Why it matters here

You'll build a hazard analysis table for the 2–3 main risks of the needle-planning
workflow (e.g. vessel puncture from a bad trajectory), each row linked back to a
requirement from Module 5 that mitigates it — closing the loop between "what could go
wrong" and "what the system does about it."

## Key terms

- **Hazard** — a potential source of harm (e.g. "trajectory crosses a vessel")
- **Harm** — the actual injury/damage that could result (e.g. "internal bleeding")
- **Hazardous situation** — circumstance where exposure to a hazard could lead to harm
- **Severity / Probability** — the two axes used to assess risk
- **Risk control / mitigation** — what the system or process does to reduce a risk
- **Residual risk** — what's left after mitigation
- **Traceability to requirements** — each hazard's mitigation should map to a
  requirement (and, indirectly, a Capella function)

## Step-by-step (high level — to be expanded)

1. Brainstorm hazards specific to the needle-planning workflow (e.g. vessel puncture,
   wrong target selected, miscalibrated trajectory display).
2. For each hazard, identify cause → effect → severity/probability.
3. Define a mitigation for each, and link it to the requirement(s) from Module 5 that
   implement it.
4. Assemble the table: hazard → cause → effect → severity/probability → mitigation →
   linked requirement ID.
