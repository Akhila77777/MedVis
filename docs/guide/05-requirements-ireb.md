## Why it matters here

You'll write a small but properly structured requirements specification for the
needle-planning tool, plus a traceability matrix linking each requirement to the
Capella function(s) from Module 4 that realize it and the check/test that verifies it.

## Key terms

- **Requirement types** — functional, non-functional (performance, usability...),
  constraint
- **Quality criteria for requirements** — unambiguous, verifiable, atomic, necessary,
  feasible (the IREB "good requirement" checklist)
- **Requirement ID / convention** — stable identifiers (e.g. `REQ-010`) used for
  traceability
- **Traceability matrix** — table linking requirements ↔ design elements ↔
  verification
- **Stakeholder vs. system requirement** — distinguishing "what the user needs" from
  "what the system must do"

## Step-by-step (high level — to be expanded)

1. Read the relevant sections of the IREB CPRE Foundation syllabus (free PDF).
2. Derive ~15–25 requirements from the operational/system analysis in Module 4 (e.g.
   "the system shall allow the user to place exactly one entry point and one target
   point").
3. Classify each requirement (functional / non-functional / constraint) and assign IDs.
4. Build a traceability matrix: requirement → Capella function → verification method
   (e.g. "manual test", "unit test").
